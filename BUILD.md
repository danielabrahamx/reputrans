## Tool Installation (Wave 0)

Install in this exact order. Each must succeed before proceeding.

### 1. Foundry

```bash
curl -L https://foundry.paradigm.xyz | bash
# Then restart shell or source ~/.bashrc, then:
foundryup
forge --version  # must succeed
```

If foundryup fails on Windows Git Bash, try:
```bash
curl -L https://foundry.paradigm.xyz -o foundryup && chmod +x foundryup && ./foundryup
```

### 2. Nargo (Noir)

```bash
curl -L https://raw.githubusercontent.com/noir-lang/noirup/main/install | bash
# Restart shell, then:
noirup
nargo --version  # must succeed
```

If noirup fails, download directly from https://github.com/noir-lang/noir/releases — get the latest stable Windows or Linux binary, place in PATH.

### 3. Barretenberg (bb)

```bash
bbup
bb --version  # must succeed
```

If bbup is not available after noirup, install via:
```bash
curl -L https://raw.githubusercontent.com/AztecProtocol/aztec-packages/master/barretenberg/bbup/install | bash
bbup
```

**CRITICAL:** nargo and bb versions must be compatible. If you get version mismatch errors during proof generation, run `noirup --version X.Y.Z` to pin nargo to a specific release and `bbup --version X.Y.Z` for matching bb. Check https://github.com/noir-lang/noir/releases for the latest stable pair.

## Wave 1: Parallel Build (Circuits + Contracts + Issuance)

These three can be built independently. Build all three before moving to Wave 2.

### Agent A: Noir Circuits (`packages/circuits/`)

**Goal:** Compile circuit, generate `plonk_vk.sol`.

Init:
```bash
cd packages/circuits && nargo init --name reputrans_proof
```

Files to create (full code in plan.md):
- `src/main.nr` — main circuit (see plan.md NOIR CIRCUITS section)
- `src/lib/map_to_curve.nr` — 30-constraint optimization (see plan.md)
- `src/lib/set_membership.nr` — Merkle membership proof
- `src/lib/nullifier.nr` — U2SSO nullifier
- `src/lib/credential.nr` — EdDSA credential verification
- `src/lib/range_proof.nr` — rating/trip range proofs

**Feedback gate:**
```bash
nargo compile      # must succeed with 0 errors
nargo test         # must pass
```

**Generate verifier contract:**
```bash
bb write_vk -b target/reputrans_proof.json -o target/vk
bb contract -k target/vk -o ../contracts/src/plonk_vk.sol
```

**Common failures:**
- `std::witness` is not valid Noir — use direct arithmetic instead
- `u1` type for Merkle indices: use `bool` if `u1` is unsupported in your nargo version
- EdDSA function signature: check `dep::std::eddsa` API for your nargo version — may be `eddsa_poseidon_verify` or `verify_signature`

### Agent B: Solidity Contracts (`packages/contracts/`)

**Goal:** Deploy U2SSORegistry and ReputationVerifier to Sepolia.

Init:
```bash
cd packages/contracts && forge init --no-commit
```

**Dependency on Agent A:** Can write and test `U2SSORegistry.sol` independently. Must wait for `plonk_vk.sol` before completing `ReputationVerifier.sol`.

Files to create (full code in plan.md):
- `src/U2SSORegistry.sol`
- `src/ReputationVerifier.sol` (imports plonk_vk.sol)
- `src/plonk_vk.sol` (generated by Agent A — do not write manually)
- `test/U2SSORegistry.t.sol`
- `test/ReputationVerifier.t.sol`
- `script/Deploy.s.sol`

**Feedback gate:**
```bash
forge build    # must succeed
forge test     # must pass
```

**Deploy to Sepolia:**
```bash
source ../../.env
forge script script/Deploy.s.sol \
  --rpc-url $SEPOLIA_RPC_URL \
  --private-key $PRIVATE_KEY \
  --broadcast \
  --verify
```

After deployment, append contract addresses to `.env`:
```
REGISTRY_ADDRESS=0x...
VERIFIER_ADDRESS=0x...
```

**Common failures:**
- `plonk_vk.sol` not yet generated: build U2SSORegistry first, wait for circuits to complete
- Gas estimation failure: add `--gas-estimate-multiplier 200` to forge script
- Verification failure: add `--etherscan-api-key` or skip `--verify` flag (verification is optional)

### Agent C: Threshold EdDSA Issuance (`packages/api/src/lib/threshold.ts`)

**Goal:** Real 3-of-5 threshold EdDSA credential signing.

This is TypeScript implementing the ThetaCrypt paper's threshold scheme.

**Key packages:**
```bash
pnpm add @noble/curves @noble/hashes
```

**Implementation pattern:**

```typescript
// threshold.ts — Real threshold EdDSA on BN254
// Paper: ThetaCrypt (https://arxiv.org/pdf/2502.03247)
// Math: Shamir secret sharing over BN254 scalar field + EdDSA partial signatures

import { jubjub } from '@noble/curves/jubjub';
// Note: BN254 EdDSA uses Baby Jubjub curve (the embedded curve of BN254)
// Baby Jubjub is the correct curve for EdDSA inside BN254 circuits
// @noble/curves does not export bn254 EdDSA directly — use baby jubjub

// Shamir secret sharing: split master signing key into n shares, reconstruct from t
// Each signer holds one share and produces a partial EdDSA signature
// Combine t partial signatures to get full threshold signature

export interface SignerShare {
  index: number;   // 1..n
  share: bigint;   // secret share of master private key
  publicKey: { x: bigint; y: bigint };
}

export interface PartialSignature {
  signerIndex: number;
  r: { x: bigint; y: bigint };
  s: bigint;
}

export interface ThresholdCredential {
  attributes: CredentialAttributes;
  signature: { r: { x: bigint; y: bigint }; s: bigint };
  groupPublicKey: { x: bigint; y: bigint };
  threshold: { t: number; n: number };
}

export interface CredentialAttributes {
  rating: number;       // e.g. 48 = 4.8 stars
  tripCount: number;
  platformId: number;   // 0x00 = Uber, 0x10 = Airbnb
  derivedKey: bigint;   // H(master_secret, platform_secret)
}

// Generate n key shares from a master signing key using Shamir (t-of-n)
export function generateKeyShares(masterKey: bigint, t: number, n: number): SignerShare[]

// Each signer computes their partial signature
export function partialSign(message: bigint, share: SignerShare): PartialSignature

// Combine t partial signatures into a full threshold signature
export function combineSignatures(partials: PartialSignature[], t: number): { r: { x: bigint; y: bigint }; s: bigint }

// Issue a threshold credential (orchestrates the above)
export async function issueThresholdCredential(
  attrs: CredentialAttributes,
  signerShares: SignerShare[],
  threshold: number
): Promise<ThresholdCredential>
```

**IMPORTANT — Curve selection:**
Noir's `dep::std::eddsa` uses Baby Jubjub (the embedded curve of BN254). Use `@noble/curves/jubjub` for TypeScript EdDSA to match. Do NOT use a different curve — the circuit will reject the signature.

**Feedback gate:** Unit tests verifying:
1. Key splitting and reconstruction via Lagrange interpolation
2. Partial signature + combination produces a valid Baby Jubjub EdDSA signature
3. Combined signature verifiable by the circuit's public key

## Wave 2: Backend API (`packages/api/`)

**Goal:** Express server exposing the full user journey as REST endpoints.

**Dependencies:** All of Wave 1.

Init:
```bash
cd packages/api
pnpm init
pnpm add express viem @noble/curves @noble/hashes
pnpm add -D typescript @types/node @types/express ts-node
```

**Endpoints:**

| Method | Path | What it does |
|--------|------|--------------|
| POST | `/identity/register` | Generate master secret, commitment, Merkle proof, submit to chain |
| GET | `/platform/demo-data` | Return hardcoded Uber demo data (rating: 4.8, trips: 1547) |
| POST | `/credential/issue` | Run 3-of-5 threshold signing, return credential |
| POST | `/proof/generate` | Generate UltraPlonk proof via noir_js |
| POST | `/proof/verify` | Submit proof to ReputationVerifier on Sepolia |

**Proof generation in Node.js (no system bb required):**
```bash
pnpm add @noir-lang/noir_js @aztec/bb.js
```

```typescript
import { Noir } from '@noir-lang/noir_js';
import { UltraHonkBackend } from '@aztec/bb.js';
import circuit from '../../circuits/target/reputrans_proof.json';

const backend = new UltraHonkBackend(circuit.bytecode);
const noir = new Noir(circuit);

const { witness } = await noir.execute(inputs);
const proof = await backend.generateProof(witness);
```

**Note:** `@aztec/bb.js` may be `@aztec/bb` depending on version. Check npm for latest package name. If UltraHonkBackend is unavailable, use `BarretenbergBackend` from `@noir-lang/backend_barretenberg`.

**Merkle tree implementation (`lib/merkle.ts`):**
- Use `@noble/hashes` Poseidon implementation, or implement Pedersen hash using `@noble/curves`
- Match the hash function used in the Noir circuit exactly
- Tree depth: 10 (1024 leaves)

**Contract interaction (`lib/ethereum.ts`):**
```typescript
import { createWalletClient, createPublicClient, http } from 'viem';
import { sepolia } from 'viem/chains';
// Load ABIs from packages/contracts/out/ after forge build
```

**Feedback gate:** `tsc --noEmit` passes, each endpoint tested with curl.

**Goal:** Next.js 15 app with 5 pages showing the full user journey.

Init:
```bash
cd packages/frontend
pnpx create-next-app@latest . --typescript --tailwind --app --no-src-dir --no-import-alias
```

**Pages (App Router):**

| Route | File | Purpose |
|-------|------|---------|
| `/` | `app/page.tsx` | Landing — problem statement + CTA |
| `/register` | `app/register/page.tsx` | Step 1: Create master identity |
| `/connect` | `app/connect/page.tsx` | Step 2: View demo Uber data |
| `/credential` | `app/credential/page.tsx` | Step 3: Threshold credential issuance (live signer panel) |
| `/prove` | `app/prove/page.tsx` | Step 4: Set thresholds, generate ZK proof |
| `/verify` | `app/verify/page.tsx` | Step 5: On-chain verification result |

**Design requirements:**
- Clean, dark theme (#0f0f0f background, white text)
- Each step clearly numbered
- Show cryptographic details (commitment hashes, signer indices) — judges will look
- Privacy split panel on verify page: "Insurer learned" vs "Insurer did NOT learn"
- Show performance metrics: proof generation time, gas used, circuit constraints

**API calls:** All pages call the local backend API (`http://localhost:3001`).
Use `NEXT_PUBLIC_API_URL=http://localhost:3001` in `.env.local`.

**State management:** Simple React state + `useState`. No Redux. Pass state between pages via URL params or sessionStorage.

**Feedback gate:** All 5 pages render, full flow completes without errors.

## Fallback Strategies

| Blocker | Fallback |
|---------|----------|
| `nargo compile` fails | Check nargo version, pin to latest stable release. Check Noir syntax changes between versions. |
| `bb contract` produces incompatible verifier | Regenerate with `--oracle_hash keccak` flag. Check bb/nargo version match. |
| Proof generation OOM in Node.js | Increase Node heap: `NODE_OPTIONS=--max-old-space-size=4096` |
| Sepolia RPC rate limited | Use `https://rpc.sepolia.org` as fallback |
| `forge deploy` runs out of gas | Wallet has 0.06 ETH — sufficient. If still failing, use `--gas-limit 5000000` |
| Baby Jubjub signature rejected by circuit | Verify public key encoding matches circuit's expected format (compressed vs uncompressed) |
| `@aztec/bb.js` API changed | Check `@noir-lang/backend_barretenberg` as alternative |
| Frontend can't reach API | CORS — add `cors()` middleware to Express, set `Access-Control-Allow-Origin: *` |
